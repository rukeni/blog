# 리액트 8기 코드 숨 1주차 회고록


## 코드숨에 들어오기까지

저는 그동안 제 코드에 자신을 가지기가 어려웠습니다. 개발일정을 여유있게 받아본 경험이 없었기에 구현위주로 작업을 했습니다.
구현은 어찌해왔지만 항상 마음속에 남아있는 코드에 대한 찜찜함을 남겨두고 다음 프로젝트를 진행했습니다.

작년 7월에 연습삼아 넣었다가 처음으로 유니콘회사 중 하나의 면접을 보았고 탈락하기는 했지만 그분이 남긴 기초적인 물음이 제게 아픔을 남겼습니다.
> 승민님이 스스로 깊숙히 알고있다고 생각하는게 있다면 자유롭게 말씀해주세요.

다른 면접 질문들은 아는 대로 잘 말하였지만, 이 물음만큼은 답변할 수 없었습니다.
뭔가를 깊이있게 탐색해보겠다고 생각해본적도 없고 한적도 없었기 때문입니다.

별 질문이 아니었지만 그 후, 제가 올해 3월까지 프리랜서생활을 하는동안 종종 마음을 아프게하고 머리속을 울리는 목소리였던 것 같습니다.
계속 고민하다가 한번쯤은 실력있는 개발자로 불려보고 싶고 나 스스로에게 자신감을 가지고 싶어서 코드숨에 문을 두드렸습니다.
이번 기회로 한번 스스로에게 자신감을 가질 수 있고 스스로 다음스텝으로 나아갈 수 있는 개발자가 되고 싶습니다.
잘부탁드립니다!

## 코드리뷰 생애 첫 시작

제가 한번도 `리뷰`라는 것을 받아본 적이 없어서 첫 코멘트가 달려서 코드숨 레파지토리로부터 과제에대한 메일이 날라왔을때, 심장이 터질뻔했습니다.(코드 리뷰를 받아본적이 없기도 했거니와 디코에서 서로 주고받는 대화가 너무 유익하셔서 겁을 너무 먹었습니다)
> 대충 막 했다고 생각하면 어쩌지?

> 너무 가혹한 평가를 받아서 앞이 막막해지면 어떡하나...(쓸데 없는 고민이었지만..)

쫄깃한(?) 심장을 잘 추스리고 열게된 윤석님의 첫 코멘트는

> 위에 빈 줄 하나를 추가해서 적절히 구분해 주면 좋을 것 같아요.

"다행히 코드 리뷰라는 것도 사람이 하는거구나" "왜이렇게 겁먹었지?"하는 생각이 들면서 긴장이 풀린 것 같습니다.

## 과제하고나서 들었던 나의 성찰

강의를 들으면서도 비슷한 생각이 들었던 것 같습니다.

뭔가 유튜브에서만 보던 분을 강의로 만나니 신기하기도 했고 천상계개발자는 뭔가 암산 탁탁해서 이거지하면 끝나는 줄 알았는데 상당히 직관적인 루틴을 따르고 있어서 이해하기 좋았습니다.

과제2번 계산기 풀이에서 완성된 calculate()함수를 보면서 저는 저렇게 심플하게는 단박에 생각하지 못할 것이라는 생각이 들었고

제가 짠 코드에서 저렇게 심플하게 가려면 많은 노력이 필요할 것이라고 생각이 들었습니다.

나름대로 뭔가 생각해서 과제를 했고 과제 내용물에 대해 '이정도면 어느정도 잘 해낸게 아닐까?' 생각했지만 막상 과제풀이를 보니까 창피했던 것 같습니다.

제가 고민했던 것보다 훨씬 심플하게 짤 수 있는 방법이 있다는 것에 감탄을 하였고 구현에만 급급한 개발만 하다가 자기 코드의 퀄리티를 높이려는 노력은 하면 할수록 큰 차이를 내겠구나라는 생각이 들었습니다.
